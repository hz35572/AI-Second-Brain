# AI Second Brain UI 参考文档（供 AI 前端开发工具使用）

> 目标：用**可实现**、**可复用**、**可对齐**的方式描述 AI Second Brain 的桌面 Web UI（灵感：ChatGPT / Perplexity）。  
> 范围：三列式对话页 + 知识库管理页 + 关键状态（空状态 / 上传中）。

---

## 1. 设计系统与视觉风格

### 1.1 风格
- 极简、整洁、专业；信息密度适中、留白充足。
- 优先使用清晰层级（标题/正文/辅助信息）而非装饰性图形。
- 交互反馈轻量：hover / active / focus-visible 清晰但克制。

### 1.2 配色（Design Tokens）
> 建议以 CSS 变量或 Tailwind theme 的形式落地。

| Token | 颜色 | 用途 |
|---|---|---|
| `--bg` | `#FFFFFF` | 页面主背景 |
| `--bg-subtle` | `#F9FAFB` | 次级背景（面板/输入区/列表底色） |
| `--text` | `#111827` | 主文本 |
| `--text-muted` | `#6B7280` | 次级文本（时间戳/辅助说明） |
| `--border` | `#E5E7EB` | 分隔线/描边 |
| `--primary` | `#4F46E5` | 强调色（主按钮/高亮操作） |
| `--primary-weak` | `#EEF2FF` | 轻强调背景（用户气泡/选中态底色） |
| `--highlight` | `#FEF08A` | 引用段落高亮 |

状态色（可按现有组件库/体系选择，保持一致）：
- 成功：绿色（用于“已完成”对勾/成功徽章）
- 危险：红色（用于“失败/删除”）
- 警告：琥珀/黄色（用于“处理中/注意”）

### 1.3 排版
- 字体：Inter（或几何无衬线替代，如 system-ui）。
- 建议字号层级：
  - 页面标题：18–20px / semibold
  - 区块标题：14–16px / semibold
  - 正文：14–15px / regular（行高 1.5–1.7）
  - 辅助信息：12–13px / regular
- Markdown 排版要点（AI 回复区域）：
  - 标题、列表、代码块、引用块、表格可读性优先。
  - 段落间距舒适（8–12px）。

### 1.4 间距与圆角
- 主要内边距：`16px–24px`
- 面板/卡片间距：`12px–16px`
- 圆角：卡片与按钮 `8px–12px`
- 分隔：使用 `1px` 细边框或浅阴影（优先边框，阴影克制）

## 2. 布局：三列式桌面 Web 应用

### 2.1 全局布局
- 左侧边栏：固定宽度 `260px`
- 中间主面板：自适应占满剩余空间
- 右侧源文档抽屉：宽度 `380px`，可折叠（默认隐藏，点击引用/来源卡片时滑出）
- 三列之间使用细边框 `--border` 分隔；整体背景使用 `--bg`，面板区域可用 `--bg-subtle`。

推荐布局结构（语义化命名）：
- `AppShell`
  - `Sidebar`（260px）
  - `ChatPanel`（flex: 1）
  - `SourceDrawer`（380px, collapsible）

## 3. 左侧边栏（导航与知识库）— 260px

### 3.1 顶部品牌区
- Logo 文案：`AI Second Brain`
- 图标：大脑图案（单色线性图标风格）

### 3.2 主操作按钮：新建对话
- 全宽按钮，主色 `--primary`，文字白色。
- hover：略微加深或增加阴影；active：轻微按压。

### 3.3 模块：知识库
- 树/列表形式展示文件夹与文件（可折叠）。
- 每项支持：图标（文件夹/文件类型）、名称、选中态（`--primary-weak` + 左侧细条或描边）。

### 3.4 模块：近期对话
- 列表项：对话标题（单行截断）+ 时间戳（`--text-muted`）。
- 支持选中态高亮（与知识库一致）。

### 3.5 底部用户区
- 用户头像 + 用户名（可选）放在底部。
- 点击弹出小菜单：`设置` / `退出登录`。

## 4. 中间面板（主对话界面）— 自适应

### 4.1 顶部栏（Conversation Header）
- 显示当前对话标题（可编辑：点击进入编辑态 / Enter 保存 / Esc 取消）。
- 右侧图标按钮：分享 / 导出（仅图标，hover 显示 tooltip）。

### 4.2 主区域（消息流）
- 可滚动；新消息出现自动滚动到底部（用户手动上滚时停止自动滚动）。

#### 用户消息气泡
- 右对齐。
- 背景：`--primary-weak`（浅靛蓝），文字 `--text`。
- 圆角：10–12px；最大宽度建议 70–80%。

#### AI 回复区块
- 左对齐；背景 `--bg`（或透明，依赖整体底色）。
- 内容用 Markdown 渲染；段落间距舒适。

##### 行内引用标记（硬约束）
- AI 回复中每个事实句/要点都应带引用标记：例如 `[1]`、`[2]`。
- 视觉形式：小型上标徽章（sup badge），可点击。
  - 默认：浅灰描边 + 深色文字
  - hover：描边/底色加深
  - active/selected：使用 `--primary` 或 `--primary-weak` 明确选中
- 点击行为：
  1) 打开右侧 `SourceDrawer`
  2) 定位到对应引用片段并高亮（`--highlight`）

### 4.3 来源卡片（AI 回复下方）
- 位于每条 AI 回复内容下方（或按引用分组展示）。
- 每张来源卡包含：
  - 缩略图（文档封面/文件类型图标）
  - 文档标题
  - 匹配摘要片段（1–2 行，关键字可轻高亮）
- 点击来源卡：打开 `SourceDrawer` 并跳转到首个匹配高亮。

### 4.4 输入区域（Composer）
- 固定在底部（sticky）。
- 大型多行输入框，占位符：`针对您的知识库，提出任何问题...`
- 输入框右侧/内部包含：
  - 附件按钮（回形针图标）
  - 发送按钮（主色按钮，disabled 状态明显）
- 输入框上方：范围选择器（见 4.5）。

### 4.5 范围选择器（Scope Selector）
- 位置：输入框上方，分段控件样式。
- 选项：
  - `全局搜索`
  - `选定文档`（附下拉：选择文件夹/文件）
- 交互：
  - 切换分段立即影响下一次提问的检索范围。
  - 当选择了特定文件夹/文件时，在控件上清晰展示名称（可截断）。

## 5. 右侧面板（源文档查看器）— 380px，可折叠

### 5.1 展示方式
- 滑出式抽屉（drawer），默认隐藏；点击引用标记或来源卡片时显示。
- 抽屉与主内容之间有明确分隔（边框/阴影二选一，优先边框）。

### 5.2 顶部栏
- 文档标题（可截断，hover 展示完整）。
- 关闭按钮（X）。

### 5.3 文档预览区
- 可滚动；展示结构化预览（PDF/Word 可先用文本抽取 + 页码信息的 MVP 形式）。
- 被引用段落使用 `--highlight` 高亮（`#FEF08A`）。
- 高亮应尽量贴合段落范围；MVP 优先保证“页码正确 + 高亮近似可见”。

### 5.4 匹配项导航
- “上一匹配项 / 下一匹配项”箭头按钮，跳转到前/后高亮区域。
- 显示当前匹配序号（可选，如 `2 / 7`）。

### 5.5 底部元数据
- 上传日期、文件大小、标签（标签用胶囊样式展示）。

## 6. 知识库管理页面（全屏视图）
> 入口：左侧边栏的“知识库”模块。

### 6.1 顶部操作栏
- 面包屑：`所有文件 > 研究 > 2026`（可点击返回上级）。
- 主操作按钮：`上传`（下拉：上传文件 / 上传文件夹）。
- 辅助按钮：`新建文件夹`、`多选`、`视图切换`（网格/列表）。
- 搜索栏：含筛选（文件类型 / 标签 / 日期 / AI 处理状态）。

### 6.2 文件网格 / 列表视图

#### 网格视图（默认推荐）
- 响应式网格卡片。
- 卡片内容：
  - 大型文件类型图标或缩略图
  - 文件名（单行截断）
  - AI 两行摘要（灰色文本）
  - AI 自动标签（彩色胶囊，例如 `#研究` `#法律` `#会议`）
  - 底部：文件大小 + 上传日期
- 悬停态：显示操作按钮（打开 / 删除 / 重命名 / 移动 / 更多）。

#### 列表视图
- 表格列：名称、AI 摘要、标签、添加日期、文件大小、处理状态。
- 行 hover 高亮；支持排序（可选）。

### 6.3 文件夹结构与快速入口
- 文件夹卡片：文件夹图标 + 名称 + 项目数量。
- 支持嵌套文件夹与拖放移动。
- 侧边栏快捷区：`与我共享`、`收藏夹`。

### 6.4 处理状态指示器
- 处理中：进度环或 `处理中...` 徽章（警告色）。
- 已完成：绿色对勾（成功色）。
- 失败：红色错误图标 + 重试入口。

### 6.5 右键上下文菜单
- 打开、重命名、移动至文件夹、删除、下载、复制链接、重新生成 AI 摘要、编辑标签。

### 6.6 空文件夹状态
- 大型空文件夹插图
- 提示文案：`将文件拖放到此处，或点击“上传”以开始使用`
- 主区域提供拖放上传区

## 7. 需视觉化设计的关键状态

### 7.1 空状态（对话页）
- 中间面板显示欢迎界面（简洁插图 + 标题 + 说明）。
- 提供建议提示语（可点击填入输入框）：
  - `总结我的研究论文`
  - `会议纪要中关于第三季度预算的内容是什么？`

### 7.2 上传状态（知识库页/上传弹窗）
- 显示拖放区（支持点击选择文件）。
- 显示上传进度条/进度环。
- 状态文本：`向量化中...`（或更细分：上传中/解析中/向量化中）。
- 失败分支：错误提示 + `重试` 操作入口。

---

## 8. 交互与可用性约束（实现要求）
- `focus-visible` 必须清晰（键盘可用），按钮/分段/菜单均可 Tab 导航。
- 所有图标按钮需 tooltip 或可访问性 label。
- 引用点击到抽屉定位：应有滚动定位与短暂强调（高亮/闪烁一次，避免持续动画）。
