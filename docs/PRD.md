# 产品需求文档 (PRD)：AI Second Brain (AI 第二大脑)
**文档版本**：V1.0
**最后更新**：2026年4月
**产品状态**：需求评审阶段

---

## 1. 产品概述和目标用户

### 1.1 产品概述
“AI Second Brain” 是一款基于大语言模型（LLM）与检索增强生成（RAG）技术打造的个人知识库问答助手。针对目前市面上知识库系统**“使用门槛高、检索不准确、查找效率低”**的痛点，本项目致力于将传统的“关键词搜索”升级为“即问即答、溯源准确”的智能交互体验。用户只需把资料扔进知识库，即可用自然语言与其对话，真正实现知识的“无缝存储与高效提取”。

### 1.2 目标用户与场景
*   **重度知识工作者**（律师、分析师、产品经理）：需要快速从大量行业报告、PDF、会议纪要中提取核心数据和观点。
*   **科研人员与学生**：面临海量文献阅读，需要跨文档串联知识点，要求引用必须严谨准确。
*   **内容创作者**：平时收集了大量素材，需要在创作时快速通过模糊记忆找回相关灵感片段。

## 2. 详细的功能列表

### 2.1 输入层（数据接入）
*   **多格式文件上传**：支持主流文档格式的上传与自动解析，包含纯文本（TXT、Markdown）及富文本/排版文档（PDF、Word、Excel等）。
*   **多模态输入支持**：除了文本，支持图片（通过OCR提取文字）、音频/视频（通过ASR模型自动转写为文本）输入。
*   **断点续传与批量上传**：支持拖拽式批量上传，并提供上传进度反馈。

### 2.2 管理层（知识整理）
*   **基础文件管理**：提供类传统网盘的管理能力，支持多级文件夹创建、重命名、移动、删除等 CRUD 操作。
*   **AI 自动标签**：文档上传并解析后，AI 提取关键词，自动为文档打上分类标签（Tag），降低用户手动整理成本。
*   **AI 自动摘要**：在文件列表中，为每个文档生成 50-100 字的极简摘要，帮助用户快速判断文档内容。

### 2.3 处理/交互层（问答引擎，核心）
*   **自然语言问答**：支持用户以大白话提问，系统通过 RAG 技术检索库内信息并生成回答。
*   **多轮对话上下文**：AI 需具备记忆能力，用户可针对上一轮的回答继续追问或要求细化。
*   **严格的引用溯源（Citation）**：AI 的每一句总结或数据提取，必须附带来源引用的角标（如：[1][2]）。点击角标可直接跳转到原文对应段落，并高亮显示。
*   **限定范围回答**：用户可以进行“全局库问答”，也可以勾选“特定文件夹/特定文档”进行局部问答，避免无关信息的干扰。

## 4. 界面设计要求 (UI/UX)

本产品的交互核心在于**“降低心智负担”**，整体采用经典的 **“Chat 优先” + “左右分栏”** 布局。

*   **极简视觉**：采用类似 ChatGPT 或 Perplexity 的极简设计，突出对话框与内容，弱化工具栏。
*   **三栏式布局**（针对 Web/桌面端）：
    *   **左侧导航栏**：知识库、历史对话记录。
    *   **中栏主对话区**：超大输入框，自然语言对话展示流。
    *   **右侧阅览区（抽屉式）**：当用户点击“溯源角标”时，右侧平滑弹出原始文档的预览窗口，并精准滚动、高亮对应段落。
*   **交互反馈**：文件解析和向量化过程中，需有清晰的骨架屏或进度条（如“正在向量化：85%”），缓解用户等待焦虑。

## 5. 非功能性需求（NFR）

### 5.1 性能要求
*   **极速响应（问答）**：流式输出（Streaming），问答的首字响应时间（TTFB）必须 **< 2秒**。
*   **高效处理（解析）**：常规文本/PDF文件（10MB 以内），从上传到完成向量化建库的时长需 **< 15秒**。
*   **大文件支撑**：支持单文件高达 100MB（约 500 页 PDF）的解析与存储，且不引起系统崩溃。

### 5.2 数据安全与隐私（产品生命线）
*   **双版本策略**：
    *   **云端版 (SaaS)**：所有传输采用 HTTPS 加密，存储的原始文件及向量数据进行 AES-256 加密，并签署且严格遵守《不用于大模型训练协议》。
    *   **本地化版 (Local)**：提供一键安装包，LLM 与 向量数据库全部在本地机器运行，完全断网可用，实现 100% 物理级数据隔离。
*   **权限管理**：系统需具备细粒度的文件权限体系（私密/公开），即便在未来加入分享功能，分享链接也需支持阅后即焚或密码保护。

### 5.3 准确性要求
*   **低幻觉容忍度**：系统提示词（System Prompt）需严格限制：*“仅根据检索到的上下文回答，如果上下文中没有信息，必须直接回答‘知识库中未找到相关内容’，严禁自行编造。”*
